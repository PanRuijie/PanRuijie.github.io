Java基础(01)--Java语言的基础思想

发表于 2019-09-12 更新于 2019-09-21 分类于 java基础知识
本文字数： 3.3k 阅读时长 ≈ 3 分钟

学习一门语言，从学习其设计思想开始

一、Java语言的面向对象思想

1.1 面向对象与面向过程

Java是一种严格的面向对象的语言，其体现在每一个.class文件都必须有一个同名公共类，所有的实现都是基于类的。传统的C语言是一种面向过程的语言，即为了实现一个目标，我们将该目标分割成一个一个的小步骤，每个步骤用一个或者多个函数去进行实现

举例而言，假如我们想要烹饪一道菜，我们需要的函数可能是准备食材、准备炊具、炒菜、装碟等等，只需要依次调用这些函数就可以实现目标。但是随着我们的系统越来越复杂，不同目标的相互关系也越来越复杂，面向过程的语言无法满足我们的设计需要了。例如我们现在是一家酒店的厨房系统，可能需要烹饪若干道菜，不同菜品之间有着这些那样的关系：有的菜品是其他菜品的辅菜，有的菜品是其他菜品的升级版。另外由于采购的原因，可能我们对于同一道菜在基础食材不一样的情况下烹饪方法有细微差别，这些功能特性只有面向对象的语言才能满足我们的需求

从本质上来说，面向对象的语言是将现实中的事物抽象成一个一个的对象，每个对象会有自己的属性和方法，属性描述的是对象的状态，方法描述的是对象的能力。想要实现一个目标，就是构造出一个个与该目标相关的对象，然后调用对象的方法进行实现。

总结面向对象与面向过程的区别与联系：

  • 面向过程：性能上优于面向对象，并且模块化程度更高更容易理解
  • 面向对象：可维护性、可扩展性更强

1.2 面向对象的三大特性

封装、继承和多态是面向对象的三大特性，Java作为严格的面向对象的语言也是具有这三大特性的。以下基于Java语言的自身特点对三大特性进行讲解

1.2.1 封装

封装：将对象实现的细节隐藏起来，然后通过一些公用的方法来暴露该对象的功能

封装的好处就是将程序使用者与程序开发者分离。程序的使用者不需要清楚的知道该程序的实现细节和一些不必要的属性，只需要知道输入输出是什么（专业术语为API）即可。这些对于程序使用者来说减少了工作量，对于程序开发者来说屏蔽了程序开发者对程序的改动，使得代码更加安全

在 Java中，封装的实现是通过访问权限控制实现封装的，对于不同的属性和方法，用不同的访问权限关键词修饰。

Java中有三个访问权限修饰符：private、protected 以及 public，如果不加访问修饰符，表示包级可见。因此实际上一共有四个等级的访问权限控制：public > protected > default(包访问权限) > private

  • public： 所修饰的类、变量、方法，在内外包均具有访问权限；
  • protected： 这种权限是为继承而设计的，protected所修饰的成员，对所有子类是可访问的，但只对同包的类是可访问的，对外包的非子类是不可以访问；
  • default (包访问权限)： 只对同包的类具有访问的权限，外包的所有类都不能访问；
  • private： 私有的权限，只对本类的方法可以使用

一个表格展示不同权限：

public √ √ √ √
protected √ √ √ x
default √ √ x x
private √ x x x

除此之外，Java还可以通过内部类的机制实现封装，具体的内容到了内部类再进行详解

1.2.2 继承

继承是指子类拥有父类数据结构的方法和机制，这是类之间的一种关系

继承是类与类的一种关系，是一种“is a”的关系。比如“狗”继承“动物”，因为狗是动物。

在Java中，继承是通过类名标签上加extends关键字实现，继承了父类的子类可以直接使用父类除private修改的成员之外所有的成员，并且也可以重写同名方法，改造成符合子类的成员方法

注意：Java语言对类的规定是只能进行单继承，即一个子类只能有一个直接父类。而对于接口，允许多继承

1.2.3 多态

多态性体现在父类中定义的属性和方法被子类继承后，可以具有不同的属性或表现方式

继承是多态的基础。多态的实现需要三个条件：

  • 要有继承
  • 要有方法重写
  • 父类引用指向子类对象

其中最核心的就是第三条———父类引用指向子类对象。例如我们可以定义Animal a = new Dog();父类声明的是一个动物类型的变量，但却指向的是一个狗类变量，因为狗是动物，它继承了动物类，所以是允许的。如果调用方法时调用了被狗类重写的方法，那么在实现时该变量虽然是一个动物类变量，却产生了狗类变量的作用，此即为多态

二、Java语言的跨平台特性

2.1 高级语言的运行机制

高级语言按程序的执行方式可以分为编译型和解释型两种

  • 编译型语言指的是使用专门的编译器，针对特定平台（即操作系统）将某种高级语言的源代码一次性“翻译”成可被该平台硬件执行的机器码（包括机器指令和操作数），并包装成该平台所能识别的可执行性程序的格式，这个转换过程被称为“编译”（compile）。
    • 编译生产的可执行性程序可以脱离开发环境在特定的平台上独立运行（只需要运行环境不需要开发环境）。并且在编译结束之后可以通过链接其他编译好的目
  • 解释型语言是指使用专门的解释器对源程序逐行解释成特定平台的机器码并立即执行的语言。解释型语言通过不会进行整体性的编译和链接处理，它相当于把编译型语言中的编译和解释的过程混合在一起同时完成
    • 与编译型语言相比，解释型语言的运行效率更低，而且不会脱离解释器独立运行，但是跨平台比较容易，只需要提供特定平台的解释器即可，也即以牺牲程序执行效率为代价方便地实现了源程序级的移植
  • Java的运行机制比较特殊，它既不是纯粹的解释型语言也不是纯粹的编译型语言。Java的源程序必须经过编译步骤，但是不会生成特定平台的机器码，而是生成一种与平台无关的字节码（即.class文件）。这种字节码不是可执行性的，必须有Java解释器来解释执行，然后针对不同平台有对应的Java解释器
    • Java编译器：将Java源文件（.java文件）编译成字节码文件（.class文件，是特殊的二进制文件，二进制字节码文件），这种字节码就是JVM的“机器语言”。javac.exe可以简单看成是Java编译器。
    • Java解释器：是JVM的一部分。Java解释器用来解释执行Java编译器编译后的程序。java.exe可以简单看成是Java解释器

2.2 JDK、JVM和JRE

JDK 、JVM和 JRE 有什么区别？

  • JVM是 Java 虚拟机（Java Virtual Machine），只能够识别.class文件，能够将.class文件中的字节码指令进行识别并调用操作系统向上的 API 完成动作
  • JRE是Java运行时环境（Java Runtime Environment）,主要包含两个部分，JVM的标准实现和 Java 的一些基本类库。它相对于JVM来说，多出来的是一部分的 Java 类库
  • JDK是Java 开发工具包(Java Developerment Kit)，它集成了JRE和一些好用的小工具。例如：javac.exe，java.exe，jar.exe 等
  • 因此总结来说，三者是逐层包含的关系：JVM被JRE包含，JRE又被JDK包含。反映在目录结构上，就是我们下载安装一个JDK之后，“jdk版本号”这个目录就是JDK，它的子目录bin文件夹就是JVM，而JRE是bin目录加上lib目录

2.3 Java的跨平台性

Java为什么能够跨平台，实现一次编写，多处运行？

  • Java 能够跨平台运行的核心在于 JVM 。不是 Java 能够跨平台，而是Java程序运行在JVM上，针对不同平台，JVM的实现是不同的
  • Java 引入了字节码的概念，JVM 能够识别字节码，并将它们解释到系统的 API 调用。针对不同的系统有不同的JVM实现，但是同一段代码在编译后的字节码是一样的。在 Java API 层面，我们调用系统声音设备的代码是唯一的，和系统无关，编译生成的字节码也是唯一的。但是同一段字节码，在不同的 jvm 实现上会映射到不同系统的 API 调用，从而实现代码的不加修改即可跨平台运行
