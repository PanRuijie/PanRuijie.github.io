折腾笔记(01)--搭建个人博客

发表于 2019-09-12 分类于 折腾笔记
本文字数： 3.4k 阅读时长 ≈ 3 分钟

生命在于折腾，从零开始手把手教你如何搭建一个酷炫的属于自己的博客！

一、搭建个人博客

本文主要是基于Node.js框架下利用github使用hexo搭建了一个个人的静态网页博客，搭建个人博客的过程主要参考了韦阳的博客，相当于对该博客的内容进行了补充和细化

1.1 下载安装Node.js

在Node.js官网下载64位稳定版的安装包，安装时全按照默认选项点next，最后在cmd中输入npm -v和node -v出现版本号相关信息说明安装成功

如果默认源速度较慢，可以添加国内镜像源，在cmd内输入

1.2 安装Git

为了把本地的网页文件上传到github上面去，我们需要用到分布式版本控制工具————Git。

安装选项还是全部默认，只不过最后一步添加路径时选择Use Git from the Windows Command Prompt，这样我们就可以直接在命令提示符里打开git了。

安装完成后在cmd中输入git --version验证是否安装成功

1.4 安装Hexo

在合适的地方新建一个文件夹，用来存放自己的博客文件，比如我的博客文件都存放在D:\MyBlog目录下。

定位到该目录下，输入npm i hexo-cli -g安装Hexo。会有几个报错，无视它就行。

安装完后输入hexo -v验证是否安装成功。

然后就要初始化博客网站，输入hexo init初始化文件夹，接着输入npm install安装必备的组件。

这样本地的网站配置也弄好啦，输入hexo g生成静态网页，然后输入hexo s打开本地服务器，然后浏览器打开http://localhost:4000/，就可以在本地浏览博客啦

二、个性化配置

个性化配置时经常会涉及到_config.yml的修改，一般来说博客根目录下会有一个该文件，是整个博客的根目录配置文件，称之为站点配置文件，在themes文件夹下每个主题的文件夹内也会有一个_config.yml，称之为主题配置文件，请读者不要混淆。

2.1 主题切换

hexo本身提供了很多种主题供选择，我这边选择的是一种极简风格的主题nexT，切换主题在hexo种是很方便的，主要分为两步：

_config.yml对格式要求及其严格，一般格式为key: value，注意冒号后面必须有且仅有一个空格，否则使用hexo g时会报错

  1. 下载对应主题的源码，解压之后将该文件夹放入博客文件夹下themes文件夹内
  2. 修改博客根目录下_config.yml文件内的theme属性，将值修改为你放入themes文件夹内的文件夹名，必须与文件夹名完全一致，否则会报错

以下内容均是根据hexo的nexT主题进行个性化配置，并不一定保证对于其他主题同样适用
