Java并发编程(01)--详解Thread

从Thread的创建说起，深入源码解析底层原理

一、线程类

1.1 创建线程

Java使用Thread类代表线程，所有的线程对象都必须是Thread类或其子类的实例。Java可以用三种方式来创建线程：

  • 继承Thread类创建线程
  • 实现Runnable接口创建线程
  • 使用Callable和Future创建线程

创建线程的三种方式的对比：

  • 采用实现Runnable、Callable接口的方式创建多线程的优缺点：
    • 优势：线程类只是实现了Runnable接口与Callable接口，还可以继承其他类。 在这种方式下，多个线程可以共享一个target对象，所以非常适合多个相同线程来处理同一份资源的情况，从而可以将CPU、代码和数据分开，形成清晰的模型，较好地体现了面向对象的思想。
    • 劣势：编程稍稍复杂，如果需要访问当前线程，则必须使用Thread.currentThread（）方法。
  • 采用继承Thread类的方法创建多线程的优缺点：
    • 优势：编写简单，如果需要访问当前线程，则无须使用Thread.currentThread（）方法，直接使用this即可获得当前线程。
    • 劣势：因为线程类已经继承了Thread类，所以不能再继承其他父类。

Callable和Runable接口的区别：

  • 事实上，Callable功能比Runable强大，提供两个Runable没有能够提供的功能
  • 一方面，Callable能够获取线程执行结果的返回值而Runable不行
  • 另一方面，Callable允许抛出异常而Runable不行

1.2 Runable接口详解：

如果没有特殊说明，所有的源码分析都是基于JDK10

Runable其实是一个很简单的接口，里面只有一个抽象的run方法，实现了该接口表示实现该接口的类的实例对象准备被一个线程作为任务进行执行

通过继承Thread类，重写其run方法，在本质上也是实现了Runable接口，重写其run方法。因为Thread类也是实现了该接口

最后运行的方式是：调用线程的start方法，线程会在操作系统的调度下选择合适的时间执行传入的对象的run方法

1.3 Callable接口详解

1.3.1 Callable源码

Callable同样是一个非常简单的接口，其内部只有定义了一个类似于Runable的run方法的call方法，不同点是返回值为泛型V，且允许抛出异常

实际使用时，Callable一般是搭配FutureTask使用或者一个类实现Callable接口，重写call方法，作为一个任务被传入

1.3.2 Future源码

Future接口表示一个异步计算的结果，它提供了五个方法：

  • boolean cancel(boolean mayInterruptIfRunning) ：尝试着去取消异步计算的过程，当发生该过程无法被取消、该过程已经计算完、该过程已经被取消等情况时都会返回false，如果返回true表示取消成功，又有多种情况：如果是还没执行过就被成功取消，则表示从未运行过该任务，如果是正在运行中，则根据传入的变量决定是否取消该任务
  • boolean isCancelled()：判断异步计算是否在完成前被取消，如果是的返回true否则返回false
  • boolean isDone()：判断异步计算的过程是否结束而无论因何结束，比如即使是因为抛出异常或者被取消，仍然是结束了，返回true
  • V get() throws InterruptedException, ExecutionException 和 V get(long timeout, TimeUnit unit) throws InterruptedException, ExecutionException,TimeoutException：前者表示阻塞着直到获取到结果，后者表示至多等待timeout这么长时间，要么获取到结果要么抛出TimeoutException异常

1.3.3 FutureTask源码

FutureTask详解：

  • FutureTask实现了RunnableFuture接口，RunnableFuture接口继承了Runnable和Future两个接口
  • 由于FutureTask类同时实现了两个接口，所以它既可以作为Runnable被线程执行，又可以作为Future得到Callable的返回值
  • 使用方法为：
    1. 首先构建一个实现类实现Callable接口，重写其call方法，把要异步执行的任务写进去
    2. 其次实例化该实现了获得一个Callable对象A
    3. 再实例化一个FutureTask类的对象B，其传入的参数就是A
    4. 接着调用B的start方法开始执行
    5. 最后通过A的get方法得到返回值
  • FutureTask是常用的实现类
